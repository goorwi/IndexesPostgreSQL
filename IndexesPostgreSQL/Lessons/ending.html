SQL Задания: Анализ запросов с использованием индексов

Задание 1: Оптимизация запросов с использованием индексов

Цель: Оптимизировать выполнение запросов с помощью создания подходящих индексов в базе данных.

Инструкции:

  1. Выберите несколько запросов из вашего приложения или базы данных, которые могли бы быть оптимизированы с использованием индексов.
  2. Определите, какие столбцы часто используются в условиях WHERE, ORDER BY или JOIN.
  3. Создайте подходящие индексы для этих столбцов.
  4. После создания индексов выполните запросы еще раз и сравните время выполнения до и после оптимизации.

Пример:

Запрос до оптимизации:

SELECT * FROM products WHERE category_id = 5 ORDER BY price DESC;

Создание индекса:

CREATE INDEX idx_category_id ON products(category_id);

Запрос после оптимизации:

SELECT * FROM products WHERE category_id = 5 ORDER BY price DESC;

Ожидаемый результат:

  • Уменьшение времени выполнения запросов после создания соответствующих индексов.

Задание 2: Выбор оптимального индекса

Цель: Создать индекс, который максимально эффективно поддерживает заданное условие.

Инструкции:

  • Ниже приведены несколько примерных записей из таблицы "products".
  • Определите, какой индекс лучше всего подходит для оптимизации запроса, который фильтрует данные по указанному условию.
  • Создайте индекс с учетом выбранного условия.

Пример записей:

  • Продукт 1: Название - "Кофемашина", Цена - 1500, Категория - "Кухонная техника"
  • Продукт 2: Название - "Микроволновая печь", Цена - 800, Категория - "Кухонная техника"
  • Продукт 3: Название - "Фен", Цена - 200, Категория - "Бытовая техника"
  • Продукт 4: Название - "Утюг", Цена - 300, Категория - "Бытовая техника"

Условие запроса:

Выбрать все продукты из категории "Кухонная техника" с ценой выше 1000.

Ожидаемый результат:

Создание индекса, который эффективно поддерживает указанное условие и обеспечивает оптимальное выполнение запроса.

Задание 4: Создание составного индекса

Цель: Создать составной индекс, который эффективно поддерживает выполнение запроса с использованием нескольких условий.

Инструкции:

  • Ниже приведены несколько примерных записей из таблицы "employees".
  • Определите, какой полей в составном индексе лучше всего подходит для оптимизации запроса, который фильтрует данные по указанному условию.
  • Создайте составной индекс с учетом выбранного условия.

Пример записей:

  • Сотрудник 1: Имя - "Александр", Отдел - "Маркетинг", Зарплата - 3000
  • Сотрудник 2: Имя - "Елена", Отдел - "Продажи", Зарплата - 2500
  • Сотрудник 3: Имя - "Михаил", Отдел - "Маркетинг", Зарплата - 3500
  • Сотрудник 4: Имя - "Наталья", Отдел - "ИТ", Зарплата - 4000

Условие запроса:

Выбрать всех сотрудников из отдела "Маркетинг" с зарплатой выше 3000.

Ожидаемый результат:

Создание составного индекса, который эффективно поддерживает указанное условие и обеспечивает оптимальное выполнение запроса.
