Индексы с включенными столбцами в базах данных

Что такое индексы с включенными столбцами:

Индексы с включенными столбцами - это тип индексов, который включает дополнительные столбцы, помимо тех, что индексируются. Это позволяет улучшить производительность запросов, которые используют эти дополнительные столбцы.

Создание индексов с включенными столбцами:

Для создания индекса с включенными столбцами в базе данных используется оператор CREATE INDEX с ключевым словом INCLUDE. Ниже приведен пример синтаксиса создания индекса с включенными столбцами:

CREATE INDEX index_name ON table_name (indexed_column) INCLUDE (additional_column1, additional_column2, ...);

где:

  • index_name - имя индекса.
  • table_name - имя таблицы, на которой создается индекс.
  • indexed_column - имя столбца, который индексируется.
  • additional_column1, additional_column2, ... - дополнительные столбцы, которые включаются в индекс.

Задача:

Допустим, у нас есть таблица "products" с информацией о продуктах, и мы хотим ускорить поиск по столбцу "наименование" (name) и одновременно получить значения столбца "цена" (price) для отображения в результатах запроса.

Создание индекса с включенными столбцами:

Для ускорения поиска по наименованию и получения цен создадим индекс с включенным столбцом:

CREATE INDEX idx_name ON products (name) INCLUDE (price);

Запрос с использованием индекса:

Теперь, когда индекс создан, поиск по наименованию будет выполняться намного быстрее, а значения цен будут возвращены из индекса без необходимости обращения к основной таблице:

SELECT name, price FROM products WHERE name = 'Шоколад';

Результаты поиска:

В результате запроса будут выведены наименования и цены продуктов с наименованием "Шоколад" из таблицы "products".
